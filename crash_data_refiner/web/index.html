Crash Data Refinement Workflow

Quick start checklist

  1. Load the crash spreadsheet and the project boundary KMZ.
  2. Confirm the latitude and longitude columns.
  3. Run refinement.
  4. Review any likely unplaced crashes in the guided review stage.
  5. Download the refined outputs and generate the PDF report if needed.

Outputs produced

  • Refined crash data file inside the KMZ boundary.
  • Crashes Without Valid Lat-Long Data extract.
  • Coordinate Recovery Review extract for rows that still need location work.
  • Excluded Coordinate Review extract for crashes you explicitly keep out.
  • KMZ crash report for Google Earth previews.
  • Map preview in the browser and optional full PDF report.

Output Notes

Each run writes outputs into a dedicated folder under outputs/web_runs.

Rows auto-filled by the coordinate recovery pass are tagged in the refined output.

The browser review workbench lets you confirm, manually place, or exclude likely crashes without downloading the workbook.

Excluded crashes are written to a separate Rejected Coordinate Review output instead of staying in the refined data.

Approved coordinate review workbooks rerun the pipeline against the original crash data and KMZ from the selected source run.

The map report uses Leaflet tiles and needs network access to load the basemap.

PDF reports are generated from refined data unless a separate PDF data file is supplied.
